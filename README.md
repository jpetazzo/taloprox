# TaloProx

TaloProx (Talos+Proxmox) is a module to provision Kubernetes clusters
with the Talos distribution (by Siderolabs) on Proxmox hypervisors.

It is very opinionated; so it may or may not work for your use-case.
You might be able to use it for inspiration or by modifiying it to suit your needs.

## Features and assumptions

Here are our technical choices:
- we create virtual machines on a PVE (Proxmox Virtual Environment) cluster
- IP addresses and VM ids must be set ahead of time, in TF variables
  (we don't want to use DHCP or automatic id assignment)
- virtual machines should be reachable from the deployment machine
  (we use use this to apply Talos machine configurations; trigger bootstrap; retrieve kubeconfig)
- use Cilium for CNI and kube-proxy replacement
- dual stack (IPV4+IPV6)
- Proxmox CSI plugin pre-installed
- metrics-server pre-installed

## Getting started

If you're not very familiar with Terraform / OpenTofu and just
want to try this module standalone, you can do this:

1. Copy `env.demo` to `env` and edit it to put Proxmox auth information. 
  (Or configure the Proxmox provider in a different way if you prefer.)
2. Copy `demo.tfvars` to e.g. `mycluster.tfvars`, and edit it to set 
   VM ids, IP addresses, Proxmox bridge name, adjust the number of
   nodes, node sizes...
3. Load the environment, apply the configuration, retrieve outputs:
  ```bash
  . ./env
  tofu apply -var-file=mycluster.tfvars
  tofu output -raw kubeconfig > /tmp/kubeconfig
  export KUBECONFIG=/tmp/kubeconfig
  kubectl get nodes
  ```

Talos bootstrap can take a few minutes, so if `kubectl get nodes`
doesn't work right away, try again a bit later.

You can check VM console in Proxmox to see what's happening during
cluster provisioning (or later to see the basic, low-level Talos dashboard).
